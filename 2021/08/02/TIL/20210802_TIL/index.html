💬 느낀 점

  • 코딩 관련 서적을 사서 읽기 시작했다. 미친듯이 심플이라는 책과 Clean Code 책 두 권을 추천받았는데, Simple is Best라고 생각하고, 스티븐 잡스도 너무나 좋아하지만, 나의 조금이나마 더 나아진 코드를 위하여 Clean Code책을 선택하였다.

  • 1장인 깨끗한 코드와 2장인 의미 있는 이름까지 읽었다. 나는 1장에서는 그래디 부치(Grady Booch)가 말한

    깨끗한 코드는 단순하고 직접적이다. 깨끗한 코드는 잘 쓴 문장처럼 읽힌다. 깨끗한 코드는 결코 설계자의 의도를 숨기지 않는다. 오히려 명쾌한 추상화와 단순한 제어문으로 가득하다.

    라는 글을 보고 내가 그렇게 코드를 짜야겠구나 ! 라고 생각이 들었다.

  • 2장에서는 검색하기 쉬운 이름을 사용하라 라는 섹션이 있는데, 그 곳에서의 예시가

    라고 나와있는데,

    ‘내가 여지껏 써왔던 코드가 위의 코드였구나. 나만 알아볼 수 있는 코드를 사용했고 어떻게든 구현에만 포커스를 맞췄구나’

    라는 생각이 들었다. 아래 코드를 보면 어디에서 사용했는지를 몰라도 어떤 것을 표현하려고 하는구나 라고 이해가 되는 것을 보고 망치 🔨 로 머리를 한 대 맞은 듯한 느낌이였다. 책을 열심히 읽어서 더 나은 더 깨끗한 코드를 쓸 수 있도록 하자.

  • 파이썬을 처음으로 배우고 있다. 자바스크립트와는 또 다른, 유연한 언어이면서도 타입이 있는 그런 언어라는 생각이 든다. 조건문, 반복문, 함수 사용, 클래스에 대해서 오늘 배웠는데 클래스 Class부분이 특히 재미있었다.

  • 클래스란 ? 변수와 함수의 집합. 즉 변수와 함수를 가진 나만의 자료형이다.

    • __init__ 함수
      • 생성자라고 부르는 함수.
      • 클래스 변수가 생성될 때 자동으로 호출되는 함수.
      • 클래스 변수 안에 변수를 생성할 때 사용
    • 생성자란?
      • 클래스 변수가 생성될 때 자동으로 호출되는 함수
      • 클래스 내부에 정의된 변수 등을 초기화할 때 사용.
    • self
      • class가 가진 변수로 만들어준다.
      • class안에 있는 변수나 함수에 접근한다.
      • 내부에서 활용되는 함수이므로 count되지 않는다.

    이렇게 class에 대해서 배워보았는데, 아주 흥미로운 문법이라고 생각한다.
